# 99-通用-性能与测试基线

## 目标

为所有接口提供统一的“效率优先”设计约束与测试基线，避免后续实现出现：
- 过度分配（GC 抖动）
- 轮询/刷屏（事件风暴）
- 重复 IO（文件重复读取）
- 不可控并发（吞吐下降或 UI 卡顿）

---

## 1) 通用性能目标（局域网会议场景）

- **发现**
  - 新上线可见：< 1s（同网段）
  - 离线感知：stale 6s / expire 12s（默认，可配置）
- **控制通道（gRPC）**
  - P99 请求延迟：< 100ms
  - 长连接复用：避免频繁握手
- **文件传输**
  - chunk：1–4MB（默认 1MB 或 2MB），可配置
  - 进度事件：<= 10Hz（采样/合并）
  - 内存：与并发数 * chunkSize 成正比，不随文件大小增长
- **远控**
  - 信令建立：< 500ms（理想）
  - HUD 刷新：2Hz 默认，上限 10Hz
  - 输入延迟：尽量 < 50–80ms（同网段）

---

## 2) 通用工程策略（效率优先）

- **幂等**
  - 所有“创建类”接口（MessageSend/CreateTransfer/CreateSession）必须支持 `requestId` 幂等
- **背压**
  - 所有 streaming 必须自然背压（await write/read），禁止无限缓存
- **采样/合并**
  - 高频事件（进度、鼠标移动、stats）必须限频与合并
- **零/少拷贝**
  - 文件 chunk 从网络到落盘尽量一次拷贝
  - 避免把整个文件读入内存
- **连接策略**
  - 控制通道与远控信令通道应复用或共享连接池
  - 断线自动重连，带 `sinceSeq`/resume 信息

---

## 3) 通用错误码约定（建议）

- `INVALID_ARGUMENT`：参数不合法（长度/范围/对齐）
- `FAILED_PRECONDITION`：版本不兼容/状态不允许
- `PERMISSION_DENIED`：未授权/模式不允许（viewOnly）
- `UNAVAILABLE`：网络不可达/对方离线（可重试）
- `RESOURCE_EXHAUSTED`：客户端/服务端过载（需要限流）

---

## 4) 测试基线（xUnit + 集成测试）

### 4.1 单元测试（必须覆盖）

- 字段约束：长度/枚举/缺失处理
- 幂等：同 requestId 多次调用一致
- 限流/采样：进度/鼠标/stats 的频率上限
- 断点续传：offset 对齐与纠错

### 4.2 集成测试（建议最少集合）

- 发现：发布 + 浏览 + 解析（mDNS / UDP fallback 各 1 组）
- 控制通道：握手 + 事件订阅
- 文件：CreateTransfer + UploadChunks + Complete（含大文件模拟）
- 远控：CreateSession + Offer/Answer + 输入流（可先只验证信令/输入 ack）

### 4.3 性能测试（建议本地基准，不强绑 CI）

- 文件吞吐：同机回环 / 同网段两机
- 事件风暴：高频进度/鼠标输入不应导致 CPU 飙升或 UI 卡顿

---

## 5) 可观测性（调试必须）

- 日志分级：Info（关键生命周期）、Debug（协议细节）、Warn/Error（失败原因）
- 关键指标：
  - discovery peers count
  - transfer throughput/remaining
  - remote rtt/fps/bitrate
  - GC/内存峰值（可选）
