# “X传输工具”UI 设计文档（C# + Avalonia）

> 目标：在局域网内实现**快速发现**、**拖拽投递文件**（可留言）、**接收可视化**、以及**低延迟远程桌面操控**。  
> 风格：精美、酷炫、专业会议工具气质（暗色霓虹 + 玻璃拟态），同时保证可读性与操作效率。

---

## 1. 产品定位与设计原则

- **3 秒完成投递**：看到目标 → 拖拽 → 留言 → 发送。
- **结构化识别**：用户以“会场 / 片区 / 昵称”树形组织，降低查找成本。
- **可靠反馈**：传输状态与远控连接状态必须清晰、可追溯。
- **会议场景优先**：大文件、临时协作、多人同时在线、弱操作成本。
- **跨平台一致**：Windows / macOS 视觉与交互一致，遵循各平台细节差异（快捷键、窗口控制）。

## 3. 关键页面设计

## 3.1 首次启动 / 快速设置（引导）

### 目的
第一次打开即可完成必要配置：身份信息 + 默认接收目录。

### 页面结构（Step Wizard）
- **Step 1：身份信息**
  - 昵称（必填）
  - 标签（可多项）：标签 1（会场）、标签 2（片区）、标签 N（可自定义）
  - 预览：`赵小明（会场1 / A片区）`
- **Step 2：默认接收目录**
  - 路径输入框 + 选择按钮（FolderPicker）
  - “打开目录”快捷入口
- **Step 3：完成**
  - 开机自启（可选）
  - 允许局域网发现（必选，含权限提示）

### 交互要点
- 标签输入采用 **Chips/Token**（回车生成、可删除）。
- 表单校验即时提示：昵称为空、目录不可写等。

---

## 3.2 主页（局域网发现 + 拖拽投递）

### 页面布局（推荐：左右分栏）
- **左栏：用户列表（Tree）**
  - 标题：其它用户（显示在线数）
  - 树结构：会场 → 片区 → 用户
  - 每个用户节点显示：
    - 在线状态点（绿/黄/灰）
    - 昵称
    - 标签摘要（hover 显示完整）
    - 设备类型/系统（可选图标）
  - 顶部工具条：
    - 搜索框（按昵称/标签模糊）
    - 刷新/重新发现按钮
    - 视图切换（树 / 平铺卡片，可选）

- **右栏：投递面板（Drop Zone）**
  - 大型拖拽区域：`拖拽文件到右侧目标，或拖到左侧指定用户/标签`
  - 最近投递（最近 10 条）
  - 当前队列（进行中/失败/完成）

### 拖拽投递交互（核心）
支持两种拖拽方式：
1) **拖到“用户节点”**：发送给该用户。  
2) **拖到“标签节点”（会场/片区）**：发送给该标签下所有在线用户（群发）。

#### 拖拽状态反馈（必须酷炫且清晰）
- 当拖拽进入可投递目标：
  - 目标节点高亮（霓虹描边 + 背景微光）
  - 右侧出现浮层提示：`发送到：会场1 / A片区（共 12 人）`
- 不可投递目标（离线用户、权限不足等）：
  - 禁止光标 + 红色提示条原因

#### 放下文件后（留言弹窗/抽屉）
放下即弹出 **留言框**（Modal 或 Right Drawer）：
- 展示文件列表（支持多文件，显示大小、类型 icon）
- 留言输入框（可多行，支持快捷键发送）
- 目标摘要（用户/标签 + 在线人数）
- 发送按钮（主按钮）与取消
- 高级选项（折叠）：
  - 是否覆盖同名
  - 是否压缩（可选，后续）

#### 发送过程与结果
发送后在右侧队列显示：
- 进度条（百分比/速度/剩余时间）
- 状态：排队中/传输中/已完成/失败（可重试）
- 群发：显示“成功 x / 失败 y”，可展开明细

## 3.3 收件箱（接收记录 + 留言展示）

### 页面结构
- 顶部筛选：
  - 全部 / 未读 / 今日 / 大文件（>500MB）等
  - 搜索（文件名/发送者/留言）
- 列表区（ListView / DataGrid）
  - 列：时间、发送者（昵称+标签）、文件名、大小、留言摘要、状态
  - 行展开：显示完整留言 + 文件操作
- 右侧详情（可选：双栏详情面板）
  - 文件预览信息（类型、大小、保存路径）
  - 操作：打开文件、打开所在目录、复制路径、标记已读、删除记录（不删文件/删文件需二次确认）

### 接收实时提示（Toast + 轻量音效可选）
当收到文件：
- 右下角 Toast：`收到：xxx.pptx` + `留言：“请帮忙投到大屏”`
- 点击 Toast 进入该条详情

## 3.4 远程桌面（低延迟控制）

> 需求强调：**画质清晰、极低延迟、响应快**。UI 需要把“延迟/帧率/码率/分辨率/丢包”可视化，并提供一键性能档位。

### 页面布局
- **左栏：可控设备列表**
  - 结构同“用户树”，但仅显示可远控的在线设备
  - 每台设备行：昵称、标签、系统、在线状态、是否忙碌
  - 搜索 + 收藏（常用设备）

- **主区：远控画布（Remote Canvas）**
  - 远程画面区域（保持比例，支持全屏）
  - 顶部浮动控制条（Auto-hide）：
    - 连接/断开
    - 性能档位：流畅 / 平衡 / 清晰
    - 分辨率：原始 / 1080p / 720p（按需）
    - 输入模式：鼠标/触控板模拟（mac 友好）
    - 快捷键：Ctrl+Alt+Del（Windows）、Cmd 组合提示（mac）
  - 右下角状态 HUD：
    - 延迟：xx ms
    - 帧率：xx fps
    - 码率：xx Mbps
    - 丢包：x%

### 交互细节
- 双击进入全屏，Esc 退出。
- 连接中：显示 Skeleton/渐变扫描动效 + 取消按钮。
- 断线：红色条提示 + 自动重连开关。

## 4. 视觉设计规范（酷炫但克制）

### 4.1 主题与材质
- **主基调**：深色背景 + 玻璃拟态卡片（轻模糊、低透明、柔阴影）
- **强调霓虹**：用于可交互高亮、拖拽目标、主按钮、连接状态
- **对比度**：正文文本对比度优先（避免“炫到看不清”）

### 4.2 建议色板（可在实现时转成 Avalonia 资源）
- 背景：#0B0F1A、#0F1526
- 面板：rgba(255,255,255,0.06) / rgba(255,255,255,0.10)
- 主强调：#7C5CFF（紫）、#23D5FF（青）
- 成功：#2EE59D
- 警告：#FFB020
- 失败：#FF4D6D

### 4.3 字体与排版
- 字体：Windows 使用系统默认（Segoe UI），macOS 使用 SF Pro（系统默认回退）
- 标题/正文层级清晰：
  - H1 20-22 / SemiBold
  - H2 16-18 / SemiBold
  - Body 13-14 / Regular
  - 辅助 12 / Regular

### 4.4 图标风格
- 线性图标 + 轻微发光（hover 时），避免写实
- 文件类型 icon：PPT/Excel/Word/Video/Audio/Image/Zip/Other

## 5. 组件规范（Avalonia 对应建议）

### 5.1 核心组件清单
- **SidebarNav**：导航项带图标与选中指示条
- **UserTreeView**：
  - 三层：会场/片区/用户
  - 节点模板：状态点 + 文本 + 右侧徽标（在线/离线/忙碌）
- **DropTargetAdorner**：拖拽高亮覆盖层（Glow Border）
- **MessageComposer**（Modal/Drawer）：
  - 文件列表 + 留言输入 + 目标摘要
- **TransferQueueItem**：
  - 进度、速度、剩余时间、重试
- **InboxList**：
  - 支持未读标记与快速操作
- **RemoteCanvas**：
  - 画面区域 + HUD + 控制条（自动隐藏）

### 5.2 状态体系（统一文案与颜色）
- **在线**：绿色点 + “在线”
- **离线**：灰色点 + “离线”
- **发现中**：蓝色点 + “发现中…”
- **传输中**：蓝紫渐变进度
- **失败**：红色 + “失败（点击重试）”
- **连接远控中**：青色扫描动效

## 6. 关键动效（酷炫的“效率感”）

- **拖拽高亮**：目标节点描边从 0→1 的霓虹描边动画（150–220ms）
- **按钮 Hover**：轻微发光 + 背景亮度提升（120ms）
- **Toast**：从右下角滑入 + 轻微模糊（180ms）
- **连接中**：渐变扫描条，避免占用太高 GPU（可在“动效强度”降低）

---

## 7. 文案与提示（建议）

- 拖拽区域默认文案：**“拖拽文件到目标用户/标签即可发送”**
- 留言框提示：**“给对方留句话（可选）”**
- 群发确认：当目标人数 > N（如 10）：
  - **“将发送给 12 位在线用户，确认发送？”**
- 失败原因示例：
  - “对方离线”
  - “写入目录失败（无权限）”
  - “网络中断，已自动重试 2/3 次”

## 10. 交付物与落地建议（面向 Avalonia）

- **资源系统**：将色板、圆角、阴影、字体层级统一放入 `Theme.axaml`，支持主题切换。
- **模板化控件**：用户树节点、传输队列项、Toast 统一 DataTemplate，保证一致性与可扩展。
- **拖拽体验**：对 TreeView 节点实现 Drop 目标命中与高亮装饰层（Adorner/Overlay）。
- **性能优先**：远控画布优先保证渲染效率；动效强度可调，默认中等。
