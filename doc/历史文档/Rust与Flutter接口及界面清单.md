# 「X传输工具」Rust / Flutter 接口与界面清单

本文档依据 [需求概述.md](./需求概述.md) 整理：**Rust 侧需实现的对外能力（建议通过 flutter_rust_bridge 暴露）**、**Flutter 侧需实现的逻辑接口与 UI**。具体函数签名可在详细设计阶段再定，此处以**能力边界与模块**为主。

---

## 一、总体分工

| 领域 | 主要承担方 | 说明 |
|------|------------|------|
| 本地配置与持久化 | Flutter 为主 | 昵称、多级标签、默认接收目录等；可序列化后传给 Rust 用于网络层展示/路由 |
| 局域网设备发现（`_localdesk._udp`） | Flutter（bonsoir） | 广播/浏览、mDNS 解析；Rust 可不实现 mDNS，由 Flutter 把「发现的节点信息」下发给 Rust |
| 用户列表树（会场 / 片区 / 用户） | Flutter UI + 数据模型 | 由发现结果 + 本地身份合并成树；若 Rust 侧维护一致性状态，可通过 FRB 同步 |
| 文件发送/接收与留言 | Flutter UI + Rust 传输层 | 拖拽与留言框在 Flutter；实际收发、落盘到「默认接收目录」宜在 Rust（性能、大文件、后台任务） |
| 接收端展示（文件名 + 留言） | Flutter | 列表/通知；内容来自 Rust 回调或事件流 |
| 远程桌面（采集、编码、传输、键鼠） | Rust 为主 | scrap、**系统硬件编码**（Windows 媒体栈 / macOS Core Media）、quinn、enigo；Flutter 负责发现连接对象、展示与控制 UI |
| 远程画面渲染 | Flutter | `texture_rgba_renderer`（或项目最终选定的等价方案）；像素缓冲由 Rust 写入或通过 FRB/Texture 桥接 |
| 单 exe 打包 | 构建与工程配置 | Flutter 桌面打包 + 内置 Rust 动态库/资源；与业务接口并列作为交付项 |

**单机自测**：上述接口设计时预留 **本机回环**（127.0.0.1 / 同机两实例或「模拟对端」模式），便于只有一台电脑时调试发现、传文件与远控链路。

## 二、Rust 侧需要实现的能力与建议 FRB 接口分组

以下按业务模块列出**建议在 Rust 实现并通过 flutter_rust_bridge 暴露**的接口类型（函数/异步函数/必要时 Stream）。名称仅为示意。

### 2.1 通用与配置

- **初始化/反初始化**：全局日志、运行时配置（线程池、缓存目录等）。
- **本机标识**：稳定 `device_id` / `session_id`（用于列表去重、连接握手）。
- **接收目录**：由 Flutter 设置 `default_receive_path`；Rust 校验路径可写，用于保存接收文件。
- **身份快照（可选）**：若传输协议需要带「昵称+标签路径」，提供 `set_local_profile(json 或结构化字段)`，与 Flutter 存储一致。

### 2.2 文件传输与留言

- **监听/服务**：在局域网内启动文件接收服务（具体传输载体可与远控共用 QUIC，或独立 TCP/QUIC；由实现选定，但对 Flutter 暴露统一「开始监听 / 停止监听」）。
- **发送**：`send_files`（对端地址或连接句柄、本地文件路径列表、留言文本、可选目标标签/用户 id）；支持进度回调（已发送字节/总字节）。
- **接收事件**：新任务到达时上报：`文件名`、**留言**、发送方标识、保存路径或临时路径；错误原因（磁盘满、校验失败等）。
- **大文件与会议场景**：非阻塞、可取消、断点续传（若需求后续补充）在 Rust 侧扩展。

### 2.3 与 Flutter 发现层的协作（若发现仍在 Flutter）

- **对端连接信息注入**：Flutter（bonsoir）解析出 IP、端口、实例 id 后，调用 Rust `connect_peer` / `register_discovered_peer`，供文件与远控共用。
- **可选**：Rust 侧实现与「标签/用户」路由相关的纯逻辑（例如根据标签选择广播频道），减少 Dart 重复逻辑。

### 2.4 远程桌面 — 被控端（Host）

- **会话生命周期**：`host_start` / `host_stop`；鉴权/配对令牌（与需求中的「指定用户」一致，可由 Flutter 传入 token）。
- **屏幕采集**：基于 scrap 的采集循环（分辨率、帧率、显示器选择）。
- **编码**：调用 **Windows 平台库**（如 Media Foundation 等系统 API）或 **macOS Core Media / VideoToolbox** 做硬件编码，输出与传输协议约定的码流格式（如 H.264/HEVC 等，以选型为准）；失败时的回退策略（降级分辨率、降帧率、可选软件编码路径等）在 Rust 内部处理。
- **发送**：quinn（UDP）向外推送媒体流；与客户端约定好包格式与时间戳。
- **键鼠注入**：接收客户端发来的输入事件，经 enigo 注入本机（需注意权限与安全边界）。

### 2.5 远程桌面 — 控制端（Client）

- **连接**：`client_connect`（对端地址、token）；`client_disconnect`。
- **解码与送显**：两种常见对接方式（二选一或并存，与 `texture_rgba_renderer` 最终对接方式一致）：
  - **A**：Rust 在控制端解码后在内存中产出 **RGBA 帧**，通过 FRB/高效缓冲交给 Flutter 纹理；解码优先使用 **各平台系统解码 API**（与 Host 侧硬件编码栈配套）。
  - **B**：Rust 只负责网络与**已编码码流**，Flutter 侧或独立 FFI 层解码（则 Rust 暴露「收包」接口；解码实现与 Host 侧约定一致，走系统/硬件解码路径）。
- **输入上报**：Flutter 将本地鼠标键盘事件传给 Rust `client_send_input_event`，经 quinn 发往被控端。

### 2.6 非功能性

- **线程安全**：采集、编码、网络、磁盘 IO 均在 Rust 侧隔离，避免阻塞 Flutter UI 线程。
- **日志与诊断**：连接失败、编码器失败、权限不足等结构化错误码，便于 Flutter 弹窗提示。

## 三、Flutter 侧需要实现的「接口」（逻辑层）

此处的「接口」指 **Dart 层服务/状态管理** 对外契约，而非仅 FRB。

### 3.1 配置与持久化

- **身份模型**：昵称、标签列表（有序，对应「会场 / 片区 / …」层级或扁平 tag，与 UI 树一致）。
- **本地存储**：读写用户配置（`SharedPreferences`/文件/SQLite 等）；启动时恢复。
- **目录选择**：调用系统目录选择器，写入「默认接收路径」，并同步调用 Rust `set_receive_path`。

### 3.2 发现与列表（bonsoir）

- **注册服务**：`_localdesk._udp`，payload 携带可解析的昵称、标签路径、端口或会话信息（与 Rust 监听端口一致）。
- **浏览与解析**：合并去重、超时剔除；生成「用户列表」数据结构。
- **树形聚合**：按标签层级构建 `会场 → 片区 → 用户` 树（与需求示例一致）；本地用户与远端用户统一展示规则。

### 3.3 文件与留言业务

- **拖拽源**：从系统拖拽文件进入应用；识别拖放目标为「某标签节点」或「某用户节点」。
- **发送流程**：校验文件可读 → 调 Rust `send_files` → 展示进度/失败。
- **留言 UI**：拖拽完成后弹出留言框，确认后与发送任务绑定。
- **接收展示**：订阅 Rust 接收事件，在界面展示 **文件名 + 留言**（列表或通知）。

### 3.4 远程桌面（Flutter）

- **会话 UI**：发起远控、断开、全屏、画质/码率档位（若暴露给 Rust **硬件编码器**参数）。
- **纹理渲染**：`texture_rgba_renderer` 与 Rust 帧输出的对接（注册 texture、每帧更新）。
- **输入捕获**：在远控窗口聚焦时捕获指针/键盘，转为协议事件经 Rust `client_send_input_event` 发出。
- **与 bonsoir 联动**：从列表选择「哪台机器的远控服务」再建立 Rust 连接。

### 3.5 单机测试支持

- **调试模式**：例如「模拟对端」、本机第二端口、或启动两个实例时的端口配置，保证发现与传输可在同一台 PC 上验证。

## 四、Flutter 侧需要实现的 UI 界面

### 4.1 启动 / 设置

- **身份设置页**：昵称输入；动态增删标签（标签 1…n）；保存。
- **接收目录设置**：显示当前路径；「选择文件夹」按钮；路径合法性提示。

### 4.2 主界面

- **个人信息区**：展示「昵称（标签1/标签2/…）」与需求示例一致。
- **用户列表区**：树形控件（可折叠）——「其它用户 → 会场 → 片区 → 用户」。
- **传输入口**：列表项/标签节点作为拖放目标（视觉高亮、禁止拖放提示）。

### 4.3 发送与接收反馈

- **留言对话框**：文本输入、发送、取消。
- **传输进度**：可选进度条或轻量提示。
- **接收记录**：文件名 + 留言 + 时间 + 发送方；可清空或导出（按需）。

### 4.4 远程桌面

- **远控入口**：从用户节点打开「远程桌面」。
- **远控窗口**：大区域纹理显示；工具栏（断开、全屏等）；可选网络状态指示。

### 4.5 通用

- **错误与权限**：目录不可写、远控权限、防火墙提示等。
- **关于/版本**（可选）：便于会议场景排障。

## 五、与《需求概述》中技术路线的对应关系

| 需求中的技术 | 承担方 | 备注 |
|--------------|--------|------|
| bonsoir（`_localdesk._udp`） | Flutter | 设备发现；Rust 专注传输与媒体 |
| scrap | Rust | 屏幕采集 |
| Windows 库 / macOS Core Media 硬件编码 | Rust | 被控端编码；Windows 侧用系统媒体 API，macOS 侧用 Core Media / VideoToolbox 等；码率分辨率等与 Flutter 设置联动可选 |
| quinn | Rust | 媒体与/或控制信道（可与文件通道分离或统一） |
| texture_rgba_renderer | Flutter | 显示路径与 Rust 输出格式对齐 |
| enigo | Rust | 被控端键鼠 |
| flutter_rust_bridge | 双方 | 配置、文件、远控、帧数据、事件回调的统一边界 |
